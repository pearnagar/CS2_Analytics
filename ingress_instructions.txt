CS2 Analytics — Enabling NGINX Ingress on Minikube
====================================================

1. Enable the ingress addon:
     minikube addons enable ingress

2. Wait for the ingress controller pod to be Running:
     kubectl get pods -n ingress-nginx --watch
   (Ctrl+C once ingress-nginx-controller-* shows STATUS = Running)

3. Deploy the whole stack via Helm (this is now the only deploy path — the old
   standalone k8s/ manifests were retired once the chart at charts/cs2-analytics/
   took over):
     helm upgrade --install cs2-analytics ./charts/cs2-analytics

4. Confirm the Ingress picked up an address:
     kubectl get ingress cs2-ingress

5. No hosts-file editing needed. The ingress hostnames in values.yaml are sslip.io
   addresses (e.g. cs2.192-168-49-2.sslip.io) that encode the Minikube IP directly
   and resolve automatically over public DNS — just browse to
   https://cs2.192-168-49-2.sslip.io and https://api.192-168-49-2.sslip.io/api/stats
   (the browser will warn about the self-signed cert — expected for local dev;
   see the mkcert note below to remove that warning too).

   If your Minikube IP isn't 192.168.49.2, run `minikube ip` and update
   ingress.frontendHost / ingress.apiHost / monitoring.ingress.*Host in
   charts/cs2-analytics/values.yaml to match (replace dots with dashes in the IP),
   then regenerate charts/cs2-analytics/templates/tls-secret.yaml with a cert
   whose SAN list covers the new hostnames before re-running the helm upgrade.
